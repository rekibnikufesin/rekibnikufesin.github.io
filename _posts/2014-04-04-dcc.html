As I mentioned during the presentation- the code on GitHub won't be actively maintained, it's for reference only. It's build off of the current MEAN Stack git (as of this writing) and it's far too easy to clone that stack and setup the pieces you need, rather than trying to recreate the wheel. In summary- we did 5 things:

  1. Clone the current MEAN Stack from git</li>
  2. Setup a Facebook app
    1. Created a developer account at https://developers.facebook.com</li>
    2. Created a new app inside of Facebook to get an api access key and secret</li>
    3. Entered the details from step #2 into the clientid/client secret fields of config/env/development.js</li>
      </ol>
      </li>

    4. Created a server side URL that allows us to download the posts for a newsfeed for a logged-in user
      1. The controller is built in app/controllers/fbfeeds.js</li>
      2. The route is built in app/routes/fbfeeds.js</li>
        </ol>
        </li>

      3. Created a UI that allows a logged-in user to see existing posts, and create new posts</li>
      4. Created a chart using d3.js to visualize our user data
        1. The chart is rendered using an Angular directive (views/reports/ppu.html)</li>
        2. D3 is loaded as a service and injected as a dependency (public/services/d3.js)</li>
        3. The d3 visualization is built inside of the directive (public/js/directives.js)</li>
          </ol>
          </li>
          </ol>
          Also, the data for the visualization came from a Mongo Aggregation query. It's possible that we could have gotten the same results using a group by operator in Mongo, but doing this route allowed us to explore implementing the aggregation framework inside of the MEAN Stack. One other reason to choose this route (possibly more important) is the mongodb groupby operator will not function correctly in a sharded environment. Therefore, while it would have worked in this demonstration, it will not scale with you as you grow.
